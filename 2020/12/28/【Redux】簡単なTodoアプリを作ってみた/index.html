【Redux】簡単なTodoアプリを作ってみた

はじめに

 今回は、Redux で作った簡単な Todo アプリについてまとめます。

 最近、筆者は Redux を学習しているんですが、なかなか難しいと感じています。そこで、知識の整理のつもりで簡単な Todo アプリを作ってみました。とても簡単なアプリながら、Redux の基本的な機能が詰まったものになっていると思います。

 当ブログ『Yuuuki Blog』では、筆者自身の学習のアウトプット、復習用メモの作成を目的に運営しています。

完成品の共有・ディレクトリ構成

 今回作った Todo アプリの外観は以下になります。非常に簡素な見た目です。

 この Todo アプリの機能は以下の 2 つです。

  • テキストボックスに内容を入力し、Add ボタンを押すと、Todo アイテムとして登録される。
  • Todo アイテムの右の Delete ボタンを押すと、Todo アイテムが削除される。

 この Todo アプリのディレクトリ構成は以下のようになります。（一部省略）

todo-redux
  ├─ node_modules
  └─ src
    ├─ componenrs
    │   └─ ItemInput.jsx
    ├─ redux
    │   ├─ store.jsx
    │   ├─ action.jsx
    │   └─ reducer.jsx
    ├─ index.jsx
    └─ App.jsx

 Redux を導入したアプリを作るとき、ディレクトリ構成ってどうすればいいのか疑問です。ducks パターンとか re-ducks パターンとか聞くけどさ

 次項から、Todo アプリのソースコードを見ていきたいと思います。

Store の作成

 アプリ全体の state の管理場所となる Store を store.jsx 内に作成します。

store.jsx

 redux 内のcreateStoreを利用し、Store を作成します。このとき、createStore の第一引数として後述する Reducer を渡します。

 createStore の第二引数には、Chrome 拡張機能である Redux DevTools を利用するためのものを渡します。この拡張機能は Redux 公式チュートリアルでも利用されており、非常に便利なものです。

  • Redux DevTools
  • 導入方法

Action の作成

 Action とは、アプリ内に起こる出来事を表すプレーンなオブジェクトのことです。この Action を action.jsx に記述します。今回は追加、削除機能を実装するための Action を作成します。この Action は必ず、type プロパティを持ち、その値は文字列となります。

action.jsx

 Action の payload は、Reducer で state を作成するための材料という解釈が個人的にはしっくりきます。

Reducer の作成

 Reducer を reducer.jsx に記述します。Reducer とは、現在の state と action から、新たな state を作成し、返す関数です。

reducer.jsx

 state の初期値 initialState を定義し、Reducer 関数の第一引数に渡しておきます。Reducer 関数の中では、switch 文を利用し、Action の type によって処理を分岐させます。

 新しい state を作る際は、いろいろなやり方があると思いますが、スプレッド構文を使用するのが、スマートだと思います。

Provider の設定

 React コンポーネントから作成した Store に アクセスできるように、react-redux 内の Provider を利用します。これは、index.js 内に記述します。このとき、import した store を渡しまます。

index.jsx

Selector と Dispatch について

 作成した Store 内の items にアクセスするために、Selector を使います。Selector は react-redux のuseSelectorで呼び出します。

 また、Redux では、state の変更方法として、dispatch 関数を利用して、Action を Store 内の Reducer へ渡すというやり方があります。dispatch は、 react-redux のuseDispatchで呼び出します。

動作確認

 実際に動作を確認してみると、上手く動作していることが Redux DevTools で確認できます。
