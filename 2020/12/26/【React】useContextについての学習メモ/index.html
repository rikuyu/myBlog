【React】useContextについての学習メモ

はじめに

 今回は、React Hooks の useContext についてまとめたいと思います。

  現在筆者は、Redux を学んでいます。そこで、state の管理方法として useContext も理解しておくべきだと考えたので、勉強メモ作成がてら記事にまとめたいと思います。

目次

  • コンテクストと useContext
  • 実際に使ってみる
  • ひとこと
  • 参考

コンテクストと useContext

 React には、コンテクストというものがあります。このコンテクストとは、コンポーネントツリー内で、親から子へとバケツリレーのようにデータを渡すことなく、簡単にデータを受け渡すことを可能にするものです。

 useContext とは、このコンテクストを利用し、簡単なデータの受け渡しを実装するための Hooks の機能ですね。

実際に使ってみる

 では、実際に useContext を使用してみたいと思います。今回作業するディレクトリは以下のような構造をしています。（一部省略）

usecontext-sample
  ├─ node_modules
  └─ src
    ├─ componenrs
    │   ├─ A.jsx
    │   ├─ B.jsx
    │   └─ C.jsx
    └─ App.jsx

 この記事では、以下のようなデータの受け渡しを useContext によって実装したいと思います。

各コンポーネントのソースコードを見てみます。

App.jsx

 App.jsx では、必要なものをインポートし、データの受け渡しの際に利用するコンテクストをcreateContextを用いて作成します。（コンテクストオブジェクトを作ります。）

 次に、今回受け渡すデータ boyInfo を useState で定義しておきます。

 最後に、App.jsx の子コンポーネントである A.jsx をコンテクストオブジェクト boyContext の Providerder でラップします。このとき、受け渡すデータ boyInfo を、value として記述します。

 これで、データを受け渡す準備が整いました。

A.jsx

 A.jsx では、B.jsx をインポートします。

B.jsx

 B.jsx では、C.jsx をインポートします。

C.jsx

 C.jsx では、必要なものをインポートし、App.jsx から useContext で、コンテクストオブジェクト boyContext そのもの を受け取ります。

 実際にブラウザで確認してみると、データの受け渡しに成功していることが確認できます。

 useContext を利用すると、C.jsx は、A.jsx, B.jsx を経由せず、App.jsx からデータを受け取ることができるので非常に便利ですね。

以上で終わります。

ひとこと

 現在 Redux を学んでいますが、なかなか難しいと感じています。冬休みを Redux の学習に使いたいと思います。
