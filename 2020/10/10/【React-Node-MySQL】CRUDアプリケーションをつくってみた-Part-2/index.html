注意事項

 この記事で紹介する CRUD アプリケーションは単に CRUD の機能を実装したものです。

 筆者自身のインプットの質を高めることを目的の一つとして、記事を執筆しています。そのため、アプリの実用性は皆無です。

 この Part 2 では、インスタンス化、待ち受け等の説明は省略します。CRUD の機能をメインにまとめます。

 筆者の僕自身プログラミング初学者のため、見苦しいコードも散見されると思います。その場合、Twitter にてご指導いただけると幸いです。

使用する DB・TABLE

 今回、DB として MySQL を利用しています。また、使用するテーブル vegetables は以下のようなものです。id・name・color の 3 カラムを持ちます。

index.js について

 まずは、モジュールをロードします。

 express、mysql、body-parser、cors を require メソッドでロードします。

 次に、Express から MySQL に接続します。（一部伏字を使用しています。）

これより、CRUD の各機能の説明に入ります。

Create

 post メソッドを使用します。Body-Parser を利用し、フロントから送信されてきた野菜アイテムの情報を取得します。

 次に、SQL の INSERT 文を定義します。（詳しい使い方は略）

 最後に、query メソッドで SQL クエリを実行し、野菜アイテムを DB に登録します。

Read

get メソッドを使用します。

SELECT 文を定義します。

 query メソッドで SQL クエリを実行し、DB からデータを取得します。

Update

  put メソッドを使用します。Body-Parser を利用し、フロントから送信されてきた野菜アイテムの情報を取得します。

 SQL の UPDATE 文を定義し、query メソッドで実行し、野菜アイテムの情報を更新します。

Delete

 リクエストされたパスから、パラメータの id を取得します。

 SQL の DELETE 文を宣言し、query メソッドで実行し、野菜アイテムを DB から削除します。

以上で、CRUD 機能の説明を終わります。
