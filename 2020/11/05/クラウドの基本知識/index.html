クラウドのメリット

 クラウドのメリットとして以下の 3 つがあることを学びました。

  • 経済性・・・ゼロから開発する必要がなく、必要なリソースを必要な期間だけ利用できるため、運用コストを最小限に抑えることが可能である
  • 柔軟性・・・リソースが不足したら拡張し、リソースが余ったら縮小するなどスケーラブルな運用が可能であり（拡張性があり）、柔軟に利用できる
  • 可用性・・・クラウドは事業者に一任しているため、クラウドを導入したシステムが災害や故障などの障害で機能しなくなることがなく、安心・安定してシステムを運用できる

クラウドの ３つのサービス提供形態

SaaS（Software as a Service）

  • ソフトウェアをサービスとしてユーザに提供する形態
  • メリットとしては、導入の速さが挙げられます。アカウントを登録するだけで利用開始ができるものが多いです。
  • Microsoft Office 365 が有名です。

PaaS（Platform as a Service

  • ミドルウェア、OS などのアプリケーションを実行するための環境をサービスとして提供する形態です。
  • メリットとしては、比較的安価なコストで環境を構築可能であり、開発にすぐに取り掛かることができることが挙げられます。
  • Azure App Service、Google App Engine、Heroku などが有名です。

IaaS（Infrastructure as a Service）

  • メモリ、ストレージ、ネットワークなどのハードウェアをサービスとして提供する形態です。
  • メリットとしては、自由に環境が開発できることが挙げられます。（その分専門的な知識が要されますが）
  • Amazon Elastic Computer Cloud（Amazon EC2）、Azure 仮想マシン（Virtual Machines）、Alibaba Cloud が有名です。

クラウド実現のために技術

 クラウドの実現のために欠かせない技術に仮想化技術があります。仮想化技術とは、物理的なハードウェアのリソース（メモリ、ストレージ、CPU など）を論理的に扱えるようにする技術のことです。

ここでは、二つのサーバー仮想化技術（ハイパーバイザー型・コンテナ型）を紹介します。

その前に、仮想サーバー実現に役立つ 3 つの機能を紹介します。

  • ロードバランサー
    これは通信データ量に応じて、複数のサーバーに処理を振り分け、分散処理を可能にする機能です。個々のサーバーにかかる負荷を減らし、大量のアクセスにも対処できる便利な機能です。
  • オートスケール
    これはサーバーにかかる負荷に応じて、サーバーを追加/削除ができる機能です。
    この機能によってサーバーを効率よく利用することが出来ます。
  • 定期スナップショット
    これは定期的に仮想サーバーのデータを保存/バックアップする機能です。

ハイパーバイザー型

 ハイパーバイザー型は、ハードウェア上でハイパーバイザー（＝仮想化ソフトウェア）を動かし、そのうえで、ゲスト OS を運用する技術です。

コンテナ型

 コンテナ型は、ハードウェアのホスト OS 上で、Docker などのコンテナ管理ソフトウェアを用いて、コンテナと呼ばれるアプリケーション実行環境を複数構築できる技術です。
